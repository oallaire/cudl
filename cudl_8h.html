Detailed Description

This library is meant to make units more usable in a C code base. It takes inspiration from the C++ user defined literals. See the documentation for usage.

Author
Olivier Allaire

cudl_v_t volts = cudl_v(10);

cudl_v_t other_volts = cudl_from_mv_to_v(mvolts); // other_volts will be equal to 10 internally.
}

Macro to add an operation that has a left hand side and right side of the same type and does not change the units. For example, adding 2 variables of volts together still results in volts, but multiplying would result in square volts. To do the former, use CUDL_ADD_NO_TRANSFORM_OP.

#include <cudl.h>
#include <stdint.h>
CUDL_ADD_UNIT(v, uint32_t)
CUDL_ADD_NO_TRANSFORM_OP(v, _add, +)
static void bar(void) {
cudl_v_t v1 = cudl_v(10);
cudl_v_t v2 = cudl_v(42);
cudl_v_t result = cudl_v_add(v1, v2); // result will contain 52.
}
Parameters
_nameThe unit to add an op for. It is expected to be the same as the _name param used with CUDL_ADD_UNIT*.
_op_nameThe core name of function to add.
_opThe op to use between the 2 values.

cudl_v_t v1 = cudl_v(10);

bool result = cudl_v_eq(v1, v2); // result will contain false.
